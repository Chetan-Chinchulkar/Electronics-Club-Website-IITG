Inventory Management system using RFID and Solenoidal Lock




Problem



In Electronics Club there are various components kept in various cupboards.The problem is that the key is always with Secretary of club.Anyone who has to hire a component have to first contact to secy for keys.There is no check that one who has hire the keys has returned yet or not.Another problem was that there was absolute lack of a check system of who has hired the components.



Solution

Description



It mainly consist of three parts:-

  • 1.Scan using RFID card.
  • 2.Proper entry in the website.
  • 3.Lock system.


Scanning using RFID card:

Components: Arduino Uno,RFID card reader,Jumper wires
For running the code in arduino,RFID Library must have to be installed. It will scan the roll no.But it will be in form of rectangular array.Then we write a python program to extract roll no.(See code in github).from the array.RFID will communicate through website using R Pi.

#include <SPI.h>
#include <MFRC522.h>
 
#define RST_PIN         9          // Configurable, see typical pin layout above
#define SS_PIN          10         // Configurable, see typical pin layout above
 
MFRC522 mfrc522(SS_PIN, RST_PIN);  // Create MFRC522 instance
 
void setup() {
    Serial.begin(9600);     // Initialize serial communications with the PC
    while (!Serial);        // Do nothing if no serial port is opened (added for Arduinos based on ATMEGA32U4)
    SPI.begin();            // Init SPI bus
    mfrc522.PCD_Init();     // Init MFRC522
    mfrc522.PCD_DumpVersionToSerial();  // Show details of PCD - MFRC522 Card Reader details
    Serial.println(F("Scan PICC to see UID, SAK, type, and data blocks..."));
}
 
void loop() {
    // Look for new cards
    if ( ! mfrc522.PICC_IsNewCardPresent()) {
        return;
    }
 
    // Select one of the cards
    if ( ! mfrc522.PICC_ReadCardSerial()) {
        return;
    }
 
    // Dump debug info about the card; PICC_HaltA() is automatically called
    mfrc522.PICC_DumpToSerial(&(mfrc522.uid));
}

Website:

It consist of a dynamic website.Front end of the website is made using HTML.Back end is made using Flask. Github link is given below: https://github.com/niteshjindalxb/Electronics_Inventory/tree/master/Raspberry%20pi%20-%20Rollno%20detector/MFRC522-python-master


Lock System:

After proper entry,website will either sent ‘1’to open the lock and ‘0’to remain closed.Nodemcu will create a server to communicate between website and lock.
Components: breadboard,Nodemcu,Solenoidal lock,Lipo Battery Thus finally a full fledged Inventory Management System is Ready.

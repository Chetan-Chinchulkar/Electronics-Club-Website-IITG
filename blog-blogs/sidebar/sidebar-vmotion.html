V-Motion




“Virtual music player which plays music according to the movements of hands.It is similar to playing a tune on a keyboard or a guitar where you need to strike different chords.To ease human efforts and demonstrate the power of computer vision, we simply generate music by motion of hands in real-time using pre-set tunes at various angles. “

KEY LEARNING-



  • Depth camera
  • Image segmentation, noise blurring, finding contours, convex hull detection, finding centroid
  • Concept of threading
  • Using OpenCV and Pygame libraries


Things used in this project



  • Intel Realsense Camera
  • Opencv
  • Python
  • Pygame


Story



This project was done with this fantastic “Open Source Computer Vision Library”, the OpenCV. In this project, we have focused on depth images processing, Python2 and music player libraries pygame. OpenCV was designed for computational efficiency and with a strong focus on real-time applications. So, it’s perfect for real-time human detection and segmentation using a camera.



3 phases



To complete the project, V-Motion, we have worked on three different phases-



  • Human detection and data gathering using depth camera
  • Depth image processing
  • Integrating music with real time human movement.

Step2

processing the image feed from depth camera.The depth camera segments the human by discarding the background after certain clipping distance.Then apply noise blurring and image smoothing to remove noise from the raw input of depth camera. Next we use convex hull detection and find contours method to extract human contours from the image.After that, moments function is used to find the centroid of the human figure. Then, we detect the palm/fist of the person which serves as an indicator of the tune to play. The angle of the hands from the centroid serves as a criterion to determine the music to be played.



Step3

The masked image was divided into eight parts radially and corresponding to the different parts, non-identical musical tunes were setup. Whenever the extreme left or extreme right (palm) intersected the divided regions, the music which tallied with the region got played in the background. Since the algorithm was very responsive and played the tunes for a numerous number of times for a single detection. So we had to apply a locking mechanism that would only allow the tune to be played once disallowing the unwanted consecutive tunes. If we want to play a same tune twice then it has to be alternate i.e. the first tune should be followed by a different tune and then the desired tune will be played again. We can consider the A to be our desired tune then this should be the sequence of the music where B is any other tune different from A.

img1 = cv2.imread('messi5.jpg')
img2 = cv2.imread('opencv-logo.png')
# I want to put logo on top-left corner, So I create a ROI
rows,cols,channels = img2.shape
roi = img1[0:rows, 0:cols ]
# Now create a mask of logo and create its inverse mask also
img2gray = cv2.cvtColor(img2,cv2.COLOR_BGR2GRAY)
ret, mask = cv2.threshold(img2gray, 10, 255, cv2.THRESH_BINARY)
mask_inv = cv2.bitwise_not(mask)
# Now black-out the area of logo in ROI
img1_bg = cv2.bitwise_and(roi,roi,mask = mask_inv)
# Take only region of logo from logo image.
img2_fg = cv2.bitwise_and(img2,img2,mask = mask)
# Put logo in ROI and modify the main image
dst = cv2.add(img1_bg,img2_fg)
img1[0:rows, 0:cols ] = dst
cv2.imshow('res',img1)
cv2.waitKey(0)
cv2.destroyAllWindows()


What is GSM?

GSM is a standard developed by the European Telecommunications Standards Institute.It was created to describe the protocols for second generation(2G) digital cellular networks used by mobile phones and is now the default global standard for mobile communications.A GSM module is a chip or a circuit that will establish communication between a mobile device and a GSM system.

Its functions include:

  • Read,write and delete SMS messages
  • Send SMS messages
  • Monitor signal strength
